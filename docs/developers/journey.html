Guide for WoT Developers

There are many ways to be an expert in the Web of Things and below is our proposal on how to get there. This assumes no prior knowledge on any of the technologies the Web of Things builds upon.

  1. JSON:
    1. Learning Sources
    2. Configuration files
    3. Exchanging data between devices (APIs)
    4. Metadata (e.g. API Descriptions)
  2. JSON Schema:
    1. Learning Sources
    2. Validation of JSON instances
    3. Generation of documentation, types for programming languages, UIs
  3. JSON-LD:
    1. Learning Sources
    2. Bringing context to data, specifying the meaning (semantics) of the data
  4. A protocol, HTTP/REST is recommended:
    1. Learning Sources for the HTTP protocol
    2. How one programs a server (Thing)
    3. How one uses a REST Client and how one programs one
  5. TD:
    1. Reading the spec
    2. Use a TD for a UI
    3. Interact with a Thing
    4. Existing Tooling
  6. Discovery:
    1. Introduction mechanisms
    2. Register TDs
    3. Find/Fetch TDs
    4. Existing implementations
  7. Scripting API:
    1. Idea behind how to program interactions with devices, the WoT way!
    2. Reading the specification
    3. node-wot
  8. Binding Templates:
    1. See how your knowledge in one protocol can apply to others
    2. See how other protocols work in WoT
  9. Profiles:
    1. Reduce implementation effort for your Consumers by implementing a profile
